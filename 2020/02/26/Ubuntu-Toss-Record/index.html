Ubuntu-Toss-Record

发表于 2020-02-26

写在前面

​ 因为选修课的缘故，就要用到Ubuntu 18.04 LTS来写东西。老师给的要求是VMware+Ubuntu18+OpenCV。但目前的surface用虚拟机还是太吃力，不仅是运行还是存储上都不宽裕，因此用了别的奇技淫巧。

初次尝试 live USB

​ 没错，曲线救国x1，用Ubuntu的live USB模式将Ubuntu寄存在U盘中即插即用。

​ 但是，坑实在是太多了。此文将慢慢地回忆起我遇到的一个个莫名其妙的未解之谜，并自我推断问题原因。谨以此文告诫未来的自己，不要重复做傻事。

食用方法

​ 使用 Universal-USB-Installer-1.9.9.0 配合 ubuntu-18.04.4-desktop-amd64.iso 制作启动U盘。其中，需要设置Persistent storage。让U盘变成便捷Ubuntu系统 简书的这篇文章就是入门坑了。分别给了两篇文章参考，但明明用了两种方法。

​ 而我，选了日后埋下问题的那一种。HOW-TOHow to Create a Persistent Linux Installation on a Bootable Live USB or SD Card 步骤非常简单，做完即插即用。（此时我有两个U盘，并分别测试了速度，才发现我一直在用的U盘只是个2.0的）

开始折腾

​ 自己的想法是面对一个新系统，需要解决网络、输入的问题。

  1. v2ray（科学上网）。

    ​ 这时看到了油管上 Ubuntu系统如何使用v2ray客户端 一通操作用另一个电脑下下来必要的文件用U盘导入（为什么不直接下呢？因为没有科学上网下不下来，就形成了一个出不去的闭环）。可怜的是我连文本编辑器也不会用，照着教程下载脚本文件。默默地在terminal上录入bash >curl，告诉我要apt install curl. 这时，我便学会了 apt 相关的命令，发现原来安装软件也是容易的（当然也要网络通畅）。

    ​ 将v2ray按照教程都配置好了后，也确实在运行，可我依然无法科学。（忙活几小时）继续google，找到了v2ray的GUI。Linux配置v2ray详细教程 才知道这个 Qv2ray 按照流程来，也确实看到在运行了，可老是出错，类似于 Failed to read response from … 可我百思不得其解，该做的设置也做了，可怎么就不行呢。直到我查询类似的错误信息时，发现了这个。最终看到了电脑的时间，将时间调准后，成功。 最后顺着这条思路寻找，原来：Ubuntu和Windows默认的时间管理方式不同，所以切换系统发生时间错乱（经尝试，此方法在18.04 版本中不适用）。找到另一个 How to Fix Time Differences in Ubuntu 16.04 & Windows 10 Dual Boot

    ​ 自此，科学上网的问题大部分被解决。

  2. 文本编辑器。

    ​ 这是一个过不去的坎，我以为Ubuntu的文本编辑跟txt差不多（想多了），当我用vi的时候，我竟一个字都打不出来，基本模式知道，可我用方向键时却出现AB之类的乱码，现在我找到了原因。ubuntu预安装的是tiny版本，所以会导致我们在使用上的产生上述的不便 所以要安装完整的vim。

    ​ 但在这之前，我使用的是 nano 。总算能写一点东西修改点文件了。看了 Nano简单使用 才知道原来的操作也是如此原始（甚至不会整行修改，一直按删除键）。也偶尔用Ubuntu自带的文本编辑器编辑文字，可以更直观的看到文字处理。

    ​ 还有呢，在terminal里面的复制粘贴是 Ctrl+shift+c/v。Ctrl+c是终止操作的执行。可以用上下浏览代码历史，还算是比较方便了。

  3. 输入法。

    ​ 先直奔已经成功的说，最终安装了 fcitx-googlepinyin 。在Ubuntu18.04系统中安装谷歌拼音输入法（Google Pinyin）

    ​ 一开始希望用上搜狗拼音，但出现了奇怪的问题，各种组件崩坏，顺便系统崩了一次无法开机，只能在busybox里面敲代码了（哭）。入坑文：ubuntu 18.04 LTS 安装搜狗输入法

    ​ 可能是完全卸载了ibus框架的原因导致输入法出错，装不上搜狗还只能打出繁体字，最终在我一次尝试后系统无法开机。

    ​ 从头再来。

  4. 更新源。

    ​ 这居然是较晚才知道的步骤，相见恨晚，应该第一时间把Ubuntu的更新源换为国内的，可以少很多麻烦。

    ​ ubuntu18.04 更新源：这一篇提供了很多源。

    ​ Ubuntu18.04下更改apt源为阿里云源：这一篇更细致并且有点高端，作为一个啥都不懂的萌新来说一开始不是很明白他在干嘛，但他提供了另一篇有用的文章：ubuntu sudo update与upgrade的作用及区别

  5. pip源。

    ​ 知道了源后，自然就会想到pip换源。以前不知道pip是什么，但看python相关的文章总有它的身影。Python 之 pip安装 及 使用详解

    ​ 安装OpenCV怎么都装不上呀，网络问题还是存在的。想到了切换国内源，一查也是可行的。Ubuntu16.04下配置pip国内镜像源加速安装

最终总算配置完全。

二次尝试

​ 最扫兴的事当属你花了比较多的沉没成本时，你又不得不脱离。当折腾了良久后配置完成一个运行良好的Ubuntu后，你的存储不够用了。当我执行一次update后，运行失败，排查错误，发现Ubuntu可用存储几乎没有了，查阅 清理缓存、垃圾、多余内核 也只能清理出300mb，不知不觉我的这些操作已经占用了快3g的大小。一个32g的U盘却只能装下6g的Ubuntu，有点憋屈。

​ 原因是作为启动盘U盘只能为FAT32格式，而此格式只能存放单个不超过4g的文件，而我的用户信息都存放在carsper-rw中，因此无法大于4g。

新的 live USB

​ 无奈，重新制作足够大的live USB.

​ 参考：·How to Create a Live Ubuntu USB Drive With Persistent Storage

​ ·Create a 95% Persistent Ubuntu 18.04 LTS USB! (Part 1) 💻🖥️ - YouTube

​ 这些教程就需要用到一个虚拟机了，最后权衡之下我选择了 VirtualBox。跟着视频一步步来最后没出什么问题。

唯二的问题

  1. 无法识别usb。

    ​ 官方有解决包：要安装VirtualBox 6.1.4 Oracle VM VirtualBox Extension Pack

    1. Ubuntu虚拟机无法全屏显示（或大分辨率）。[
      How To Make Ubuntu Full Screen](https://www.youtube.com/watch?v=RxmGFsaOyks)

新的之前

​ 尝试用修改U盘分区的办法，但最终失败了：How to make a persistent live Ubuntu USB with more than 4GB

​ 原因：This method does not work with 64 bit Ubuntu 14.04 and later.

​ 这也促使我知道了busybox这个东西。

最后的最后

​ 关于安装并运行一个Ubuntu 18.04 LTS 算是告一段落了。

​ 可以开始使用了 _(:з」∠)_。
